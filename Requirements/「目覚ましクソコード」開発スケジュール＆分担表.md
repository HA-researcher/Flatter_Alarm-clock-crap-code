**開発期間**: 3月3日（火）〜 3月14日（土） \[計12日間\]

## **1\. チーム体制と詳細分担（モジュール別）**

メンバーのスキルとVibe codingの活用を考慮し、以下のようにモジュールを分割・担当します。

### **🍎 ｃｃいちご (PM / Backend / ML / API)**

* **役割**: プロジェクト全体の管理、インフラ構築、API・MLの基盤実装。  
* **担当モジュール**:  
  * **インフラ/環境構築**: GitHubリポジトリ作成（2種）、Next.js・Flutterの初期化、Vercel・Supabaseのプロジェクト作成。  
  * **API接続**: `CodeGeneratorService`, `CodeReviewService` のベース実装（Next.js API Routes または クライアントからの直接呼び出し）。  
  * **機械学習**: MediaPipeによる顔・ランドマークトラッキングのベース実装。  
  * **PM**: タスクの進捗管理、全体の結合調整。ロゴ制作（余裕があれば）。

### **🍊 太田あやき (Backend / ML / Prompt Engineering)**

* **役割**: AIの「賢さ」と「クソコードらしさ」の演出、監視精度の向上。C言語や機械学習の知見を活かす。  
* **担当モジュール**:  
  * **プロンプトエンジニアリング**: Gemini APIへ送る「クソコード生成プロンプト」「コードレビュー（合否判定）プロンプト」のチューニングと精度検証。  
  * **機械学習**: ｃｃいちごが作ったMediaPipeベース実装に対し、閉眼・顔消失判定の「閾値」調整やロジックの最適化。  
  * **バックエンド**: SupabaseのDBスキーマ設計、Realtimeの動作検証サポート。

### **🍋 秋元慎太郎 (Front Web / Front Mobile)**

* **役割**: アプリケーションのフロントエンド全体のアーキテクチャ設計と、Flutterの実装。  
* **担当モジュール**:  
  * **フロントMobile (Flutter)**: 「PC＆スマホ版」におけるスマホアプリ全般。QRスキャン、Supabase Realtimeリッスン、爆音アラーム制御（マナーモード貫通など）。  
  * **フロントWeb (React/Next.js) 基盤**: プロジェクトのルーティング設定、Supabase Client・状態管理（`SessionProvider`）の実装、Monaco Editorの組み込み。

### **🍇 京角 (Front Web / Vibe Coder)**

* **役割**: Vibe coding（CursorやGitHub Copilot, Gemini等のAI活用）を駆使したWebフロントのUI/UX実装。  
* **担当モジュール**:  
  * **UIコンポーネント実装**: HTML/CSS/JSの基礎を活かし、AIに指示を出して各画面（Home, Waiting, Challenge, Monitoringなど）の見た目を作成。  
  * **演出・アニメーション**: 後輩AIのSOS演出UI、警告画面のCSSアニメーションなどの実装。  
  * ※秋元さんが作った状態管理やエディタの「枠」に、デザインやUIパーツをはめ込んでいくイメージ。

## **2\. リポジトリ構成**

ご要望に合わせて、以下の**2つのリポジトリ**で進めます。

1. **`mezamashi-kusocode-standalone`**  
   * **概要**: PCのみで完結するバージョン（Web単体モード）。  
   * **技術**: React / Next.js  
2. **`mezamashi-kusocode-sync`** (PC＆スマホ版)  
   * **概要**: スマホ連携を前提としたバージョン。1つのリポジトリ内でWeb側とスマホ側をディレクトリを分けて管理します（例: `/web` と `/mobile`）。  
   * **技術**: React / Next.js (Web側), Flutter (スマホ側)

*※初期段階では「UIコンポーネント」や「APIロジック」は単体版・連携版でほぼ共通です。まずは単体版で基本機能を動かし、その後連携版へコードを移植・拡張する進め方を推奨します。*

## **💡 Vibe Codingの進め方アドバイス**

初学者の場合、最初から「機能」を作ろうとするとエラーで詰まりがちです。以下の順序でAI（CursorやGemini）に指示を出すとスムーズです。

1. **「見た目」だけを作る（静的モックアップ）**  
   * *プロンプト例*：「React(Next.js)とTailwind CSSを使って、設定画面のUIを作って。時刻設定ピッカー、難易度のセレクトボックス、二度寝検知のトグルボタンが必要です。」  
2. **「動き（見た目上の）」をつける**  
   * *プロンプト例*：「さっきの画面で、難易度で『カスタム』を選んだ時だけ、その下にテキストエリアが表示されるようにして。」

## **⚠️ 開発上の注意点**

* **PCのカメラ権限**：MediaPipeを動かすためにはローカルホスト(localhost)またはHTTPS環境が必要です。開発中からHTTPSかlocalhostを維持してください。  
* **無料枠の制限**：Gemini APIの無料枠制限（RPM制限など）に引っかからないよう、開発中の無駄なAPIコールには注意しましょう（モックデータを活用すると良いです）。
