# **要件定義書**

## **1\. プロジェクト概要**

### **1.1 プロジェクト名**

目覚ましクソコード（仮）

### **1.2 目的・背景**

「ねむい...二度寝したろ！」を許容しない、エンジニア向けの新感覚目覚ましアプリケーションを開発する。 時間になると爆音アラームと共に、「先輩……！サーバー負荷が大変なんです……！助けてください！」と懇願する後輩AIから、課題（計算量やメモリの無駄が多い「クソコード」）が出題される。 ユーザーはPCのコードエディタでリファクタリングを行い、AI（Gemini）によるコードレビューを突破して「少しでも改善された」と認められなければ、アラームは鳴り止まない。 厳密な実行速度の計測ではなく、AIを用いたエンタメコーディング体験を提供し、朝活としての頭の体操を促進することを目的とする。

### **1.3 ターゲットユーザー**

## **2\. システム構成**

### **2.1 技術スタック（無料枠・低コスト運用）**

* **Frontend (Web):** React / Next.js  
* **Frontend (Mobile):** Flutter  
* **Backend / DB / State Sync:** Supabase (Database \+ Realtime)  
* **Code Editor:** Monaco Editor  
* **AI / Code Generation & Review:** Google Gemini API  
* **Computer Vision:** ブラウザ向け機械学習ライブラリ (MediaPipe等)  
* **Hosting:** Vercel (Web)

### **2.2 システムアーキテクチャ・リポジトリ構成**

* **リポジトリ構成:** GitHubでの「マルチレポ」構成とする。（Webフロントエンド用リポジトリと、スマホアプリ用リポジトリの2つを分けて管理する）

### **2.3 動作モードと連携・運用上の制約**

本システムは、ユーザーの環境に合わせて以下の2つの動作モードを提供する。

1. **Web単体モード（PCのみ）**  
   * PCのブラウザ上でアラーム設定、鳴動、コーディング、監視のすべてを完結させるモード。  
   * 非連携時はPC本体のスピーカーからアラーム音が鳴る。  
2. **PC・スマホ連携モード**  
   * 毎晩、就寝前にPC（Web）でセッションを作成し、スマホ（Flutter）でQRコードを読み込んで連携状態を確立する。  
   * スマホを「爆音目覚ましデバイス」として枕元に置き、解除（コーディング）はPCで行う。  
* **PCの運用制約:** Webブラウザの制限上、PCがスリープすると処理が停止するため、どちらのモードにおいても「PCは電源に接続し、スリープさせない設定にして開いたまま就寝する」ことを前提とする。

## **3\. 機能要件**

| ID | 機能名 | 機能概要 |
| ----- | ----- | ----- |
| F-01 | セッション連携 | PCで発行したQR/合言葉をスマホで読み込み、Supabase Realtimeでデバイス間の状態を同期する。 |
| F-02 | アラーム設定 | 起床時間、音量、挑戦したい言語（約32言語）、難易度レベルを設定する。PC画面には「二度寝検知機能のON/OFFトグル」を配置する。 |
| F-03 | カメラ権限の事前取得 | アラーム設定時にWebカメラの権限を要求・取得する。**ユーザーが権限を拒否した場合は、二度寝検知機能を強制OFFとして動作させる。** |
| F-04 | クソコード生成 | アラーム鳴動時、設定された言語・レベルに基づき、Gemini APIを用いて動的に「クソコード」を生成・提示する。 |
| F-05 | アラーム鳴動 | 時間になると爆音アラームが再生される。スマホ連携時はスマホから、非連携時はPCから鳴動する。 |
| F-06 | コード編集機能 | PC画面のMonaco Editor上でコードをリファクタリングする。**コーディングを開始（エディタにフォーカス等）すると、アラーム音は一時停止する。** |
| F-07 | AIコードレビュー | 修正したコードをGemini APIで判定し、「改善されたか」を評価する。 |
| F-08 | アラーム停止・再開 | AIが「合格」と判定するとアラームが完全に停止し、二度寝監視モード（ONの場合）へ移行する。 |
| F-09 | 二度寝検知監視 | PCのWebカメラで顔/目を監視。顔消失(3秒)または閉眼(10秒)を検知するとアラームが「再開」する。起きていると判定されている間は一時停止する。 |
| F-10 | 遅延起動時チャレンジ | PCがスリープしてしまい指定時刻にアラームが鳴らなかった場合でも、**事後にPCを開いた時点で「設定時刻を過ぎていて未クリア」であれば、即座に問題提示画面（遅刻ペナルティUI）に移行する。** |
| F-11 | フェールセーフ＆緊急停止 | APIエラー時や制限超過時は\*\*「後輩ちゃんのクソコードは他の人が遠隔で解決してくれたみたいです！先輩、起こしてごめんなさい～！」\*\*という専用メッセージを表示し強制停止する。また、PC・スマホ双方に「緊急停止ボタン」を常設し手動で止められるようにする。 |

## **4\. ハッカソンにおける前提条件・許容事項（開発スコープの定義）**

限られた開発期間（ハッカソン）でMVPを完成させるため、以下の事項を割り切り・許容条件として定義する。

1. **ユーザー認証の非実装:**  
   * アカウント登録やログイン機能は実装せず、手軽さと実装スピードを優先する。データはセッション（部屋）ごとに一時的に管理する。  
2. **デモの前提とチート対策:**  
   * 本番のプレゼン・デモは「事前録画」で行う想定とする。そのため、ユーザーが意図的に不正を行う「チートモード（例: エディタのコンソールから強制クリア信号を送る等）」への強固な対策は考慮しない。  
3. **想定システム負荷:**  
   * 商用レベルのトラフィックは想定せず、同時接続数は「5～10人/時」程度を想定してアーキテクチャを組む（無料枠で十分捌ける範囲）。  
4. **パフォーマンス・クライアント負荷の許容:**  
   * 二度寝検知機能において、Webカメラを用いて「1秒間に5回」の画像処理（顔・瞳検知）を行うため、稼働中のブラウザ（PC本体）にある程度のCPU/メモリ負荷がかかることをシステム仕様として許容する。

### **2.1 PC側 (Web \- React/Next.js)**

* **メイン操作画面:** セッションのホスト、QRコード表示、各種設定を行う。

* **目覚ましデバイス:** 指定時間に爆音を鳴らす。（アプリを開いたまま就寝する）  
* **状態リッスン:** Supabase Realtimeを監視し、PC側の状態（鳴動、一時停止、合格、ペナルティ）に連動して音を制御する。  
* **フェールセーフ:** スマホ画面上の緊急停止ボタン（押下時、PC側も連動して処理を終了）。

## **2\. モジュール仕様 (主要コンポーネント)**

### **2.1 Web (React) コンポーネント**

* **`SessionProvider`**: Supabaseとの接続、現在のルーム状態（waiting, alarming, coding, cleared, penalty）をグローバルステートとして管理する。  
* **`CodeGeneratorService`**: 選択された言語とレベルからプロンプトを構築し、Gemini APIを叩いてクソコードを取得するモジュール。  
* **`CodeReviewService`**: ユーザーの修正コードと元のコードを比較するプロンプトを構築し、Gemini APIに合否を判定させるモジュール。  
* **`SleepDetectorService`**: Webカメラのストリームを取得し、MediaPipe等で顔のランドマークを解析。1秒間に5回（200ms）のループで`face_missing_time`と`eye_closed_time`を計測し、閾値を超えたらステータスを`penalty`に更新する。

### **2.2 Mobile (Flutter) コンポーネント**

* **`QRScannerScreen`**: カメラを起動し、PC画面のQRコード（ルームIDとパスコード等を含む）を読み取る。  
* **`AlarmService`**: `audioplayers`等のパッケージを利用し、マナーモードを貫通（あるいは最大音量）で音を鳴らす制御を行う。  
* **`SupabaseListener`**: 対象ルームのレコード変更を検知するストリーム。`status`が`alarming`または`penalty`になったら音を鳴らし、`coding`(一時停止用)や`cleared`になったら音を止める。

# **データ仕様書 (Supabase Schema)**

## **1\. テーブル: `rooms`**

PCとスマホの連携状態、現在のアラームの進行状況、および出題設定を管理する。

| カラム名 | データ型 | PK/FK | 制約 | 説明 |
| ----- | ----- | ----- | ----- | ----- |
| id | UUID | PK | 必須 | ルームの一意なID。QRコードに含める。 |
| target\_time | TIMESTAMP | \- | 必須 | 設定された起床時刻。 |
| language | VARCHAR(30) | \- | 必須 | 選択されたプログラミング言語 (例: "python", "javascript")。 |
| level | VARCHAR(20) | \- | 必須 | 難易度の種別。`beginner`, `intermediate`, `advanced`, `custom` のいずれか。 |
| custom\_level\_prompt | TEXT | \- | NULL可 | `level` が `custom` の場合のみ使用。ユーザーが自由記述した「出してほしい問題の要望」を保存する。 |
| is\_sleep\_detection\_on | BOOLEAN | \- | 必須 | 二度寝検知機能がONかOFFか (デフォルト: true)。カメラ権限拒否時は強制false。 |
| status | VARCHAR(20) | \- | 必須 | 現在の進行状況。詳細は下記。 |
| current\_code | TEXT | \- | NULL可 | Geminiが生成した初期クソコード。 |

### **`status` カラムの取り得る値（ステートマシン）**

* `waiting`: アラーム設定完了・時間待ち状態。※PCスリープ等でこの状態のまま指定時刻を過ぎた場合、アプリロード時にフロントエンド側で判定し、強制的に `alarming` に遷移させる。  
* `alarming`: 時間到達・アラーム鳴動中（後輩AIからのSOS）。  
* `coding`: ユーザーがエディタを操作開始。アラーム音は一時停止する。  
* `reviewing`: Geminiへコードレビューのリクエスト送信中。  
* `cleared`: レビュー合格。二度寝検知ONの場合は監視モードへ移行。  
* `penalty`: 二度寝検知によるペナルティ状態（音が再開）。起きれば `cleared` または専用の休止ステータスに戻る。  
* `force_stopped`: 緊急停止ボタンが押された、またはAPIエラー等で強制終了した状態。

## **2\. セキュリティルール (RLS: Row Level Security)**

* ユーザー認証（Auth）を使用しないため、全ユーザーに対して `SELECT`, `INSERT`, `UPDATE` を許可する。  
* ハッカソン用として、悪意のある大量作成を防ぐためのレートリミット等の厳密な防御はスコープ外とする。

## **3\. 手動テスト (E2E)**

* **実機就寝テスト:** 実際に就寝前にアラームをセットし、翌朝正常に作動するかを確認する。  
* **スリープ遅延起動テスト:** PCをスリープさせ、設定時刻を過ぎてからPCを開きWebアプリをリロードした際、即座に「遅刻チャレンジ」画面に移行するか確認する。  
* **※対象外事項:** ブラウザのコンソールからSupabaseのステータスを直接書き換えてクリアする等の「チート行為」に対する脆弱性テストは、デモ録画前提のため実施しない。

# **シーケンス図 (データ通信フロー)**

PC (React)、スマホ (Flutter)、Supabase、Gemini API 間の通信と状態遷移のタイミングを定義する。

## **1\. アラーム設定 〜 待機 〜 鳴動・課題提示**

sequenceDiagram  
    participant P as PC (React)  
    participant M as スマホ (Flutter)  
    participant S as Supabase (DB/Realtime)  
    participant G as Gemini API

    %% 連携フェーズ  
    P-\>\>S: 部屋作成 (INSERT rooms: status='waiting')  
    S--\>\>P: ルームID取得  
    P-\>\>P: QRコード表示  
    M-\>\>P: QRコード読み取り  
    M-\>\>S: 対象ルームのSubscribe開始  
      
    Note over P, M: 就寝 〜 指定時刻到達

    %% 鳴動フェーズ  
    P-\>\>P: 時刻チェック (target\_time到達)  
    P-\>\>S: UPDATE rooms: status='alarming'  
    S--\>\>M: \[Realtime\] status='alarming' 検知  
    M-\>\>M: 爆音アラーム再生開始  
      
    %% クソコード生成  
    P-\>\>G: クソコード生成プロンプト送信 (言語, レベル, カスタム入力)  
    G--\>\>P: クソコード返却  
    P-\>\>P: エディタ画面にコード表示・後輩SOS演出  
    P-\>\>S: UPDATE rooms: current\_code (任意: スマホ表示用)

## **2\. コーディング 〜 AIレビュー 〜 合格**

sequenceDiagram  
    participant P as PC (React)  
    participant M as スマホ (Flutter)  
    participant S as Supabase (DB/Realtime)  
    participant G as Gemini API

    %% コーディング開始 (一時停止)  
    P-\>\>P: エディタにフォーカス (入力を開始)  
    P-\>\>S: UPDATE rooms: status='coding'  
    S--\>\>M: \[Realtime\] status='coding' 検知  
    M-\>\>M: アラーム音 一時停止

    %% 提出とレビュー  
    P-\>\>P: 「提出」ボタン押下  
    P-\>\>S: UPDATE rooms: status='reviewing'  
    P-\>\>G: レビュー用プロンプト送信 (元コード, 修正コード)  
    G--\>\>P: JSON返却 { "is\_passed": true/false, "message": "..." }

    alt 不合格の場合  
        P-\>\>P: フィードバック表示  
        P-\>\>S: UPDATE rooms: status='alarming' (または coding維持)  
        Note over P, M: statusがalarmingに戻った場合、スマホで再度音が鳴る  
    else 合格の場合  
        P-\>\>P: 合格演出表示  
        P-\>\>S: UPDATE rooms: status='cleared'  
        S--\>\>M: \[Realtime\] status='cleared' 検知  
        M-\>\>M: アラーム音 完全停止  
    end

## **3\. 二度寝検知監視 〜 完了 または ペナルティ**

sequenceDiagram  
    participant P as PC (React)  
    participant M as スマホ (Flutter)  
    participant S as Supabase (DB/Realtime)

    Note over P, M: status='cleared' かつ is\_sleep\_detection\_on=true の状態

    loop 毎秒5回 (200ms間隔)  
        P-\>\>P: MediaPipeで顔/目をトラッキング  
        alt 閉眼10秒 or 顔消失3秒 検知  
            P-\>\>P: ペナルティ発動  
            P-\>\>S: UPDATE rooms: status='penalty'  
            S--\>\>M: \[Realtime\] status='penalty' 検知  
            M-\>\>M: 爆音アラーム 再開  
        else 再び目を開けた/顔を認識した  
            P-\>\>P: ペナルティ解除  
            P-\>\>S: UPDATE rooms: status='cleared'  
            S--\>\>M: \[Realtime\] status='cleared' 検知  
            M-\>\>M: アラーム音 停止  
        end  
    end

    %% 監視終了  
    P-\>\>P: 「完全に起床した」ボタン押下  
    P-\>\>P: カメラ監視ループ終了  
    Note over P: 最終的な完了画面へ遷移
